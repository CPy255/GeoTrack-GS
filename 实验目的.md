# GeoTrack-GS：一项战略性且严谨的实验蓝图

## 1. 实验目标与叙事

本实验方案旨在构建一个逻辑严密且具有说服力的实证叙事，以证明 GeoTrack-GS 的优越性。叙事结构如下：

1. **确立竞争力：** 首先，在标准的密集视图基准上，证明 GeoTrack-GS 的性能全面达到或超越了包括其基线（FSGS）在内的所有 SOTA 方法。

2. **突出核心贡献：** 接着，在 3DGS 领域的核心挑战——"稀疏视角重建"上，展示 GeoTrack-GS 相较于所有竞争对手（包括专为此设计的 DNGaussian 和 CoR-GS）具有压倒性的鲁棒性和更高的保真度。

3. **解构创新来源：** 然后，通过严谨的消融研究，科学地量化我们从 FSGS 基线出发所添加的每一个创新模块（GF, PHR, AR, GT-DCA）的独立贡献。

4. **验证泛化性与机制：** 证明该方法的鲁棒性可泛化至"野外"真实场景，并提供定性的视觉证据来直观解释我们方法的工作机制。

5. **评估效率与局限性：** 最后，透明地评估方法的计算开销，并坦诚地分析其局限性，以符合 SCI 一区期刊的严谨性要求。

---

## 2. SOTA 基准测试：证明全面优越性

### 目的
在公认的标准（密集视图）基准上进行全面对比，证明 GeoTrack-GS 在渲染质量和几何保真度方面均代表了最先进水平。

### 2.1 在标准 NVS 基准上的渲染质量

**目标：** 在公认的前向场景 (LLFF) 和 360° 全景 (Mip-NeRF 360) 数据集上验证渲染质量。

**对比模型：**
- 3DGS (Original)
- Mip-NeRF 360
- FSGS (Ours' Baseline)
- DNGaussian
- CoR-GS
- Ours (GeoTrack-GS)

**数据集：** LLFF, Mip-NeRF 360

**指标：** PSNR (↑), SSIM (↑), LPIPS (↓)

### 2.2 在重建基准上的渲染与几何保真度

**目标：** 在黄金标准 DTU 数据集（密集视图设置）上，证明 GeoTrack-GS 在渲染和几何两方面的双重优越性。

**对比模型：**
- FSGS (Ours' Baseline)
- DNGaussian
- CoR-GS
- SuGaR
- Ours (GeoTrack-GS)

**数据集：** DTU (标准密集视图)

**指标：**
- **渲染：** PSNR (↑), SSIM (↑), LPIPS (↓)
- **几何：** Chamfer Distance (CD) (↓), F-Score (↑), Normal MAE (↓)

## 3. 核心贡献：稀疏视角重建的鲁棒性

### 目的
证明 GeoTrack-GS 的核心优势在于解决了 3DGS 研究的主要痛点：稀疏视角。本节旨在展示，在 SOTA 方法和我们基线（FSGS）都开始失效的极端稀疏条件下，GeoTrack-GS 仍能保持卓越的几何与渲染鲁棒性。

### 3.1 视角稀疏性的影响

**目标：** 量化 GeoTrack-GS 在极少输入视图下相比竞争对手的性能优势。

**对比模型：**
- FSGS (Ours' Baseline)
- DNGaussian
- CoR-GS
- Ours (GeoTrack-GS)

**设置：**
在 DTU 数据集上，使用逐渐减少的输入视图数：**2, 3, 5, 8, 12**。

**指标与可视化：**
绘制二维折线图：
- **x轴：** 输入视图数量
- **y轴：** 性能指标

每个模型绘制两条关键曲线：
- **渲染指标：** PSNR
- **几何指标：** Chamfer Distance

输出完整的性能画像，以展示 GeoTrack-GS 的性能曲线衰减最慢。

## 4. 消融研究：解构贡献来源

### 目的
科学地回答："为什么 GeoTrack-GS 在第 3 节的稀疏测试中表现出色？" 本节旨在通过渐进式添加实验，解构我们方法的贡献，证明从 FSGS 基线出发，我们提出的每一个组件都为最终的性能和鲁棒性提升做出了独立且可量化的贡献。

### 4.1 消融序列（渐进式添加）

1. Baseline (FSGS)
2. \+ Geometric Framework (GF)
3. \+ Progressive Hybrid Regularization (PHR)
4. \+ Anisotropic Regularization (AR)
5. \+ GT-DCA (Full Model)

### 4.2 消融测试平台

**数据集与设置：**
- **DTU (稀疏设置)：** 例如，固定使用 5 视图设置，以直接验证模块对稀疏鲁棒性的贡献。
- **DTU (密集设置)：** 作为补充，证明模块在标准情况下的有效性。

**指标：**
所有配置均需报告完整的渲染 (PSNR, SSIM, LPIPS) 与几何 (CD, F-Score, Normal MAE) 指标套件，以展示性能的单调提升。

## 5. 泛化性与"野外"鲁棒性

### 目的
证明 GeoTrack-GS 在受控实验（稀疏视角、消融）中所展示的鲁棒性，可以成功泛化到更具挑战性、更"非结构化"的真实世界"野外"数据集中。

**对比模型：** DNGaussian, Ours (GeoTrack-GS)

**数据集：** NeRF Extreme, ScanNet++

**输出内容：**
- **主要：** 并排对比视频，突出基线方法的漂浮物、几何扭曲和闪烁，而 GeoTrack-GS 保持稳定。
- **次要：** 定量分数 (PSNR ↑, LPIPS ↓)。

## 6. 可视化：提供机制的直观证明

### 目的
超越量化指标，提供定性的、机制性的证据，直观地展示 GeoTrack-GS 的内部组件（如 GT-DCA）为何以及如何工作，以创建更优越的几何形状并修正伪影。

### 6.1 SOTA 与消融研究的对比视觉材料

**模型：** 来自第 2.2 节与第 4.1 节的所有模型。

**输出：** 针对能突出挑战（如薄片结构、无纹理表面）的场景，生成 RGB 渲染图、深度图和法线图。

### 6.2 核心机制的"视觉证明"

- **轨迹稳定性热力图：** 展示学习几何的平滑与一致性。
- **自适应置信度图：** 显示模型识别与处理不确定性的能力。
- **高斯基元形状可视化：** 对比正则化前后基元形状（展示其更贴合表面）。
- **GT-DCA 模块"工作实况"图：** 对比 Ours-AR 与 Ours (Full Model)。在 Full Model 图上叠加高亮区域，显示 GT-DCA 模块的主动修正范围，从机制角度解释结果为何更优。

## 7. 效率与开销分析（Q1 期刊补充）

### 目的
全面量化 GeoTrack-GS 所带来的额外计算与存储开销，将其与基线及 SOTA 方法对比，以评估其在真实应用中的可行性。

### 7.1 实验设置

**硬件平台：** 在单张 NVIDIA RTX 4090 (24GB) GPU 上进行。

**对比模型：** FSGS (Ours' Baseline), DNGaussian, CoR-GS, Ours (GeoTrack-GS)

### 7.2 关键指标

| 指标 | 测量内容 |
|------|---------|
| **训练时间** | 训练 30,000 次迭代达到收敛所需的总时间（小时）。 |
| **推理速度** | 渲染 800x800 分辨率图像的平均每秒帧数（FPS）。 |
| **显存占用** | 训练和推理期间的峰值 GPU 显存占用（GB）。 |
| **模型大小** | 最终模型所包含的 3D 高斯基元数量（百万个）。 |

## 8. 局限性与未来工作（Q1 期刊补充）

### 目的
坦诚地分析 GeoTrack-GS 方法的能力边界，确保科学严谨性，并为社区的未来研究指明方向。

### 8.1 局限性讨论

1. **对基线初始化的依赖：** 我们的方法构建于 FSGS 之上。若 FSGS 提供的初始点云在极端情况下（如错误的相机位姿）完全崩溃，GeoTrack-GS 的性能也可能受到显著影响。

2. **在极端无纹理区域的挑战：** 在面对大面积、完全无纹理的表面时，仅从稀疏视图中约束几何仍是巨大挑战，可能导致表面轻微不平整。

3. **复杂表面材质的处理：** 与大多数 3DGS 方法类似，GeoTrack-GS 在处理镜面反射或透明物体时仍面临挑战。

4. **超参数的敏感性：** 我们的新模块（PHR, AR）引入了新的损失权重。尽管提供了鲁棒的默认值，但在特定场景下的最优调参可能仍需要一定努力。

### 8.2 未来工作

- 探索自适应调整正则化权重的方法，以减少手动调参。
- 将 GeoTrack-GS 鲁棒的几何约束能力扩展到动态场景和非刚性物体的重建中。
- 研究如何将材质先验知识融入框架，以更妥善地处理无纹理和高光表面。
